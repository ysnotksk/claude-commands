# Japanese Proofreading with MeCab

MeCab による形態素解析を活用して日本語テキストを校正する。

**入力**: $ARGUMENTS

## 手順

### 1. 入力の取得

- ファイルパスが与えられた場合 → `Read` で読み込む
- 直接テキストが与えられた場合 → そのまま使用する

### 2. MeCab で形態素解析を実行

```bash
echo "対象テキスト" | mecab
```

長文の場合は一時ファイルに書き出してから実行:

```bash
mecab < /tmp/proofread_input.txt
```

### 3. 解析結果をもとに以下のチェックを行う

#### 助詞の誤用・不自然な用法
- は/が、を/に、で/に 等の使い分けを前後の形態素から判定
- 格助詞と動詞の格フレームの不整合

#### 文体の混在
- です/ます体 と だ/である体 が混在していないか
- 文末表現の品詞パターン（助動詞「です」「ます」vs「だ」「である」）を集計

#### 重複表現（冗長表現）
- 「まず最初に」「一番最後」「約〜ぐらい」「返事を返す」「被害を被る」
- 「頭痛が痛い」「馬から落馬」「後で後悔」等

#### 送り仮名・活用形の誤り
- 動詞・形容詞の活用形が辞書形から想定される活用と一致するか
- MeCab が未知語として検出した語は要注意

#### 不自然な単語境界
- MeCab が予期しない位置で単語を分割している場合 → タイポの可能性
- 未知語（MeCab 出力で品詞が「名詞,固有名詞」以外の未知語）を報告

#### 長文の検出
- 1文あたりの形態素数が 40 を超える文をフラグ
- 句点「。」で文を区切り、各文の形態素数をカウント

#### 同一助詞の連続
- 「〜の〜の〜の」のように同一助詞が3回以上連続する箇所を検出
- 格助詞・係助詞ごとにカウント

#### レジスターの不統一
- 口語的表現（「やっぱり」「ちゃんと」「すごく」）とフォーマルな表現の混在
- 話し言葉の終助詞（「よ」「ね」「さ」）がフォーマル文中に出現

#### 同音異義語の誤用候補
- 以外/意外、対象/対称/大賞、保障/保証/補償、過程/仮定/家庭
- 製作/制作、回答/解答、特長/特徴、異常/以上
- 前後文脈から明らかに誤用と判断できる場合のみ指摘

### 4. 校正レポートを出力

以下のフォーマットで出力する:

```markdown
## 校正結果

### 問題点

| # | 行 | 該当箇所 | 種別 | 指摘内容 | 修正案 |
|---|-----|---------|------|---------|-------|
| 1 | L3  | 「...」  | 文体混在 | です/ます体の文中にだ/である体が出現 | 「〜である」→「〜です」 |
| 2 | L7  | 「まず最初に」 | 重複表現 | 「まず」と「最初に」が重複 | 「まず」または「最初に」 |
| ...| ... | ...     | ...  | ...     | ...   |

### 文体判定
- 検出された文体: (です/ます体 / だ/である体 / 混在)
- です/ます体の文末: N箇所
- だ/である体の文末: N箇所

### 統計
- 総文数: N
- 総形態素数: N
- 平均文長(形態素数): N
- 最長文: N形態素 (行X)
- 未知語数: N
```

### 5. 問題なしの場合

問題が検出されなかった場合は、その旨を報告し、統計情報のみ出力する。

## 注意事項

- 出力はすべて日本語で行う
- 具体的な該当箇所を引用し、行番号を明示する
- 実際の問題のみ指摘する。スタイル上の好みは指摘しない
- テキスト全体の書き換えは行わない。問題箇所と修正案のみ提示する
- MeCab の辞書は ipadic を前提とする
