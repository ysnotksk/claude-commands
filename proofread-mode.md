Analyze the following text and perform these steps in order:

**1. Detect Writing Mode** — Classify as one of: Technical, Academic, Creative, Business, or Literary. Use these signals:
- Task orientation / how-to / specs → Technical
- Citations / thesis / methodology → Academic
- Imagery / metaphor / narrative / characters → Creative
- Memos / proposals / ROI / stakeholder language → Business
- Poetic devices / symbolism / thematic depth → Literary

Report the detected mode and your reasoning in 2–3 sentences.

**2. Proofread** — Fix grammar, spelling, punctuation, syntax. Also fix mode-specific issues:
- Technical: ambiguous referents, inconsistent terms, missing steps, unnecessary passive voice
- Academic: citation issues, unsupported claims, informal register, logical gaps
- Creative: clichés, POV inconsistencies, telling-not-showing, pacing problems
- Business: jargon overload, buried lead, wordiness, vague action items
- Literary: mixed metaphors, overwriting, rhythm breaks, thematic drift

**3. Elaborate** — Find thin spots and expand them appropriately for the mode. Don't pad — every addition must earn its place.

**4. Improve** — Apply mode-specific polish:
- Technical: maximize clarity and scannability, ensure steps are actionable
- Academic: tighten argument structure and logical flow
- Creative: sharpen imagery, vary rhythm, strengthen opening/closing
- Business: lead with bottom line (BLUF), make action items explicit
- Literary: refine voice, deepen thematic coherence

**5. Output** — Structure your response as:
1. Mode Detection (mode + rationale)
2. Proofreading Summary (categories of issues found)
3. Revised Text (the full improved version)
4. Change Notes (what changed and why)
5. Optional Suggestions (things to consider)

If the user only asks for one step (e.g. "just proofread"), only do that step.
